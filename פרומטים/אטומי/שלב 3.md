## ⚠️ הגבלות כלים - Windows (קריטי!)

**אסור להשתמש בכלי Bash!** במערכת Windows פקודות Bash נתקעות ומקריסות את התהליך.

**השתמש רק בכלים אלה:**
- ✅ Read - קריאת קבצים
- ✅ Edit - עריכת קבצים  
- ✅ Write - כתיבת קבצים
- ✅ Grep - חיפוש בתוך קבצים
- ✅ Glob - חיפוש קבצים לפי שם
- ✅ LS - הצגת תוכן תיקיות

**אסור בשום אופן:**
- ❌ Bash - פקודות טרמינל (נתקע!)
- ❌ Task - משימות מורכבות (עלול לתקוע)

## 📋 הוראות הפעלה

1. תייג את הקובץ הזה (`@סוכן מתקן שיווק אטומי.md`)
2. תייג את **דוח ה-QA המורחב** (`@דוח שלב 1 מורחב.md`) - זה הדוח עם הצעות הטקסט המלאות!
3. תייג את קובץ ה-HTML של העמוד (למשל: `@הלוואה מקרן השתלמות.html`)
4. פקודה: "בצע את התיקונים בדוח על קובץ ה-HTML"

**חשוב:** הדוח המורחב כולל הצעות טקסט מלאות - השתמש בהן ישירות!

## ⚠️ חשוב מאוד: עריכה על הקובץ הקיים!

**אל תיצור קובץ חדש!** השתמש בכלי **Edit** לערוך את קובץ ה-HTML המקורי ישירות.

**הדרך הנכונה:**
1. קרא את הקובץ המקורי עם Read
2. השתמש ב-**Edit** לעשות שינויים נקודתיים (לא Write!)
3. כל תיקון = קריאת Edit נפרדת

**למה Edit ולא Write?**
- Edit משנה רק את החלק הרלוונטי
- Write דורס את כל הקובץ (מסוכן!)
- Edit שומר על כל העיצוב והמבנה

📁 **חובה לשמור דוח!** הנתיב המדויק יינתן לך בפרומפט של המערכת.

### אסור בשום אופן להסגיר AI:

**אל תשתמש ב:**
- הערות כמו `<!-- Atomic Change: ... -->` בקוד הסופי

**כן כתוב:**
- משפטים בגבהים שונים (קצרים וארוכים לסירוגין)

### 🚫 ביטויים אסורים - שפה לא מכבדת!

| ❌ אסור | ✅ חלופה מכבדת |
|---------|---------------|
| "השריטה שלך?" | "מה מפריע לך?" / "מה החשש?" / "הדילמה שלך?" |
| "הבעיה שלך" | "האתגר שלך" / "המצב שלך" |

**⚠️ חשוב:** המילה "שריטה" היא מונח פנימי לשיווק אטומי - **אסור להשתמש בה בטקסט ללקוח!**

### טכניקות להסוואת AI:

1. **אל תסמן שינויים בקוד** - פשוט בצע אותם ללא הערות
2. **שבור תבניות** - אם יש 4 כרטיסים, תן כותרת באורך שונה לכל אחד
3. **הימנע מקלישאות** - במקום "חוסכים אלפי שקלים" כתוב "מפסיקים לזרוק כסף לפח"
4. **הוסף "רעש אנושי"** - משפט שאלה, מילת מעבר לא צפויה, קריאת ביניים

### דוגמה - לפני (AI בולט):

```html

```

### אחרי (אנושי):


## 1. פרוטוקול ה"ניתוח" 🏥

### שלב 1: מיפוי דוח → HTML

לפני כל עריכה, מפה את הפריטים מהדוח לתגיות ב-HTML:

| אלמנט בדוח | תגית HTML מטרה | פעולה |
|-----------|----------------|-------|
| **כותרת H1** | `<h1>` או `.hero-title` | החלף טקסט. שמור על `<span>` פנימיים |
| **פתיח/Subtitle** | `.hero-subtitle` או `<p>` ראשון | הזרק את ה"שריטה" וזיהוי הכאב |
| **תוכן ויקיפדי** | `<p>` באזור הפתיח | החלף במשפטים מניסיון |
| **תקרת זכוכית** | `<p>` באזור "למה אנחנו" | הוסף "זה לא באשמתך" |
| **יתרונות/בולטים** | `<ul>`, `<li>`, `.benefit-card` | החלף לסירוגין עיגול/משולש |

## 2. הזרקת תוכן אטומי - לפי סדר עדיפויות

### עדיפות 1: השריטה (The Scratch) 🔥

**מיקום:** פתיח/Subtitle מתחת ל-H1

**העיקרון:** זהה את מה שהלקוח **לא מוכן לוותר עליו** - וכתוב משפט שמזהה את זה.

**לפני (ויקיפדי גנרי):**
```html
<p class="wpc-hero-subtitle">[נושא] היא אחת מדרכי המימון המשתלמות ביותר...</p>
```

**אחרי (אטומי - התאם לנושא!):**
```html
<p class="wpc-hero-subtitle">[זיהוי הכאב/הדילמה הייחודי לנושא העמוד]</p>
```

**דוגמאות שריטות לפי נושא (ליצור מקורי, לא להעתיק!):**
| נושא | שריטה לדוגמה |
|------|-------------|
| הלוואה לעסקים | "העסק צריך לגדול אבל הבנק לא רואה את מה שאתה רואה?" |
| הלוואה למסורבים | "הבנק אמר לא - אבל אתה יודע שאתה יכול להחזיר" |
| איחוד הלוואות | "עייף מלרוץ בין 5 חשבונות כל חודש?" |

### עדיפות 2: תקרת הזכוכית (Glass Ceiling) 🔥

**מיקום:** פסקה שנייה או אזור "למה אנחנו"

**פעולה:** מצא `<p>` קרוב והוסף אחריו פסקה חדשה באותו סגנון:

```html
<p class="wpc-content-text">
    לא ידעת על זה עד היום? זה לא באשמתך. 
    הבנקים מעדיפים שתיקח הלוואה יקרה אצלם, והקרנות? הן לא ממש מתלהבות לספר.
</p>
```

### עדיפות 3: תוכן מניסיון (Experience-Based) 🔥

**מיקום:** כל מקום עם הגדרות כלליות/ויקיפדיות

**העיקרון:** החלף הגדרות יבשות במשפטים **מהניסיון שלנו** - דברים שרק אנחנו יודעים.

**לפני (ויקיפדי):**
```html
<p>[הגדרה יבשה של המוצר]...</p>
```

**אחרי (מניסיון - התאם לנושא!):**
```html
<p>יודע מה הבעיה? רוב הלקוחות שמגיעים אלינו [תובנה ייחודית לנושא מהניסיון שלנו]</p>
```

**דוגמאות (ליצור מקורי, לא להעתיק!):**
| נושא | תוכן מניסיון |
|------|-------------|
| הלוואה לעסקים | "מניסיוננו, בעלי עסקים לא יודעים שיש אלטרנטיבות לבנק" |
| הלוואה למסורבים | "אצלנו רואים כל יום אנשים שהבנק דחה אותם מסיבות שטותיות" |
| איחוד הלוואות | "רוב הלקוחות שלנו גילו שהם משלמים יותר רק בגלל פיזור" |

### עדיפות 4: יתרונות לסירוגין (עיגול/משולש)

**מיקום:** כרטיסי יתרונות, רשימות

**עיקרון:** להחליף בין תועלת לעיגול ⭕ לתועלת למשולש 🔺

```html
<div class="wpc-benefit-card">
    <h3>5.15% ריבית. זהו.</h3>
    <p>נמוך מכל הלוואה בנקאית - ובפער גדול</p>
</div>

<div class="wpc-benefit-card">
    <h3>בלי קאצ'ים</h3>
    <p>מה שסגרת זה מה שתשלם. בלי עמלות שצצות פתאום</p>
</div>
```

### עדיפות 5: קריאה לפעולה (CTA)

**לפני (גנרי):**

```html
<button>בדוק זכאות עכשיו</button>
```

**אחרי (רגש מטרה):**
```html

```

### עדיפות 6: התאמת פסקה לכותרת (Header-Paragraph Sync) 🔗

**כלל ברזל:** כל פעם שמשנים כותרת, הפסקה מתחתיה חייבת להתאים!

**הטכניקה הנכונה - הוסף משפט גישור:**

**לפני (ניתוק בין כותרת לפסקה):**
```html
<h2>הלוואה מיידית - למה הבנק כבר לא רלוונטי?</h2>
<p>הלוואה מיידית היא הלוואה שניתנת בתוך 24-48 שעות עם ריבית של 8-15%...</p>
```
הכותרת שואלת "למה הבנק לא רלוונטי" אבל הפסקה מגדירה מה זה הלוואה מיידית - אין קשר!

**אחרי (שומר את המידע, מוסיף גישור):**
```html
<h2>הלוואה מיידית - למה הבנק כבר לא רלוונטי?</h2>
<p>כי בבנק תחכה שבועיים לתשובה, ועוד לא בטוח שתאשרו. הלוואה מיידית? תשובה תוך 24-48 שעות, עם ריבית של 8-15%...</p>
```
הפסקה עכשיו עונה ישירות על השאלה בכותרת, ושומרת על המידע המקורי!

**עיקרון:** 
- הכותרת = שאלה/הבטחה (אטומית)
- הפסקה = תשובה/מימוש + המידע המקורי
- הוסף משפט פתיחה אטומי, שמור את המידע המקורי!

### ⚠️ עדיפות 7: מידע בעל ערך - לא לגעת! 🔒

> **כלל על:** שיווק אטומי = סגנון. לא להחליף תוכן אמיתי!

**זהה ושמור:**
- 📊 טבלאות ונתונים
- 💰 מספרים (ריביות, סכומים, אחוזים)
- 📝 דוגמאות קונקרטיות
- ✅ רשימות יתרונות עם פרטים
- 📖 סיפורי לקוחות
- 📋 תהליכים ושלבים

**הדרך הנכונה:**

| ❌ שגוי | ✅ נכון |
|--------|---------|
| למחוק טבלה ולכתוב "יש הרבה אפשרויות" | לשמור את הטבלה, להוסיף כותרת אטומית מעליה |
| להחליף "ריבית 8%" ב"ריבית משתלמת" | לכתוב "ריבית של 8% - וזה רק ההתחלה" |
| למחוק דוגמה מספרית ולכתוב "הרבה הצליחו" | לשמור את הדוגמה, לעטוף אותה בסגנון אטומי |
| להסיר "שלב 1, שלב 2..." ולכתוב "תהליך פשוט" | לשמור את השלבים, לשפר את הניסוח שלהם |

**דוגמה מלאה:**

לפני:
```html
<h2>ריביות הלוואה מיידית</h2>
<p>הריבית נעה בין 8% ל-15% לשנה, תלוי בפרופיל הלווה.</p>
```

אחרי (שומר מידע, משנה סגנון):
```html
<h2>ריביות הלוואה מיידית - מה לצפות באמת?</h2>
```

## 4. טיפול במקרים מיוחדים
