### א. מבנה טיפוסי של קובץ HTML בפרויקט:
```
שורות 1-100:      פתיחה + CSS Variables
שורות 100-3500:   CSS עיצוב (לדלג!)
שורות 3500-6500:  HTML תוכן (לנתח!)
שורות 6500-7500:  JavaScript + Schema (לדלג!)
```

### ב. פרוטוקול הקריאה - 4 שלבים:

**שלב 1: זיהוי מבנה הקובץ**
```
קרא את הקובץ עם:
- offset: 1, limit: 50 (לראות את הפתיחה)
- offset: 3400, limit: 100 (לאתר איפה מתחיל ה-HTML)
- offset: סוף הקובץ - 200 (לאתר איפה נגמר התוכן)
```

**שלב 2: חיפוש טקסטואלי לאיתור אזורי תוכן**
השתמש ב-grep לחפש:
```
grep "<h1" - למצוא כותרות ראשיות
grep "<h2" - למצוא סעיפים
grep "</style>" - למצוא איפה נגמר ה-CSS
grep "<script" - למצוא איפה מתחיל ה-JS
```

**שלב 3: קריאת התוכן בחלקים של 500 שורות**
לאחר שזיהית את האזור (למשל שורות 3500-6500):
```
קרא: offset: 3500, limit: 500
קרא: offset: 4000, limit: 500
קרא: offset: 4500, limit: 500
... וכך הלאה עד סוף התוכן
```

**שלב 4: צבירת ממצאים**
לאחר כל קריאה, רשום לעצמך:
- כותרות שנמצאו
- משפטים בעייתיים (ויקיפדיים/YMYL)
- דוגמאות טובות מניסיון
- אזורי CTA

### ג. מה לדלג עליו:
❌ בלוקי `<style>...</style>` - CSS לא רלוונטי לתוכן
❌ בלוקי `<script>...</script>` - JavaScript
❌ `@media` queries - עיצוב מובייל
❌ CSS class definitions
❌ Schema JSON-LD

### ד. מה לחלץ ולנתח:
✅ `<h1>`, `<h2>`, `<h3>` - כותרות
✅ `<p>` - פסקאות תוכן
✅ `<li>` - רשימות יתרונות/נקודות
✅ `<strong>`, `<b>` - טקסט מודגש (בולטים)
✅ `<button>`, `<a class="...btn">` - קריאות לפעולה
✅ תוכן בתוך divs עם class שמכיל "content", "text", "section"

### ה. דוגמה לתהליך:

1. הפרוטוקול האטומי ⚛️

עליך לבדוק האם התוכן מכיל את המרכיבים הבאים:

א. הקהל הכפול (עיגול מול משולש)
- העיגול ⭕: רוצה להצליח, אופטימי, "הילד הנבחר".
- המשולש 🔺: ספקן, מפחד מהמחיר/סיכון, "הילד המורד".
*כלל ברזל:* אם מדברים רק לעיגול - המשולש יחסום. אם מדברים רק למשולש - העיגול ישתעמם.

ב. השריטה (The Scratch)
- האם זוהה על מה הלקוח *לא מוכן לוותר*?
- האם הפתרון (B2) מכבד את השריטה הזו?

**דוגמאות שריטות בעולם ההלוואות:**
- "רוצה הלוואה אבל לא מוכן להתחנן בבנק"
- "צריך כסף אבל לא מוכן לחתום על ערבויות מטורפות"
- "רוצה לדעת מה האופציות אבל לא מוכן שימכרו לי משהו"
- "צריך מימון אבל לא מוכן שיחפרו לי בהיסטוריה"

ג. שינוי האמונה ב-4 שלבים (המפה)

**דוגמאות מעולם ההלוואות:**
- A1 (השקר): "אם הבנק אמר לא, אין לי סיכוי"
- A2 (האמת): "הבנק בודק לפי קריטריונים נוקשים שלא מתאימים לכולם - יש גופים אחרים עם קריטריונים שונים"
- B1 (האשליה): "צריך לתקן את האשראי במשך שנים לפני שאפשר לקבל הלוואה"
- B2 (הגאולה): "יש גופים שמסתכלים על תזרים ולא רק על היסטוריה"

ד. מבנה CPBG (משולש תוכן אטומי)
- טענה (Claim - האמונה החדשה).
- הוכחה (Proof - היגיון/נתונים/מקרה בוחן **מהניסיון שלנו**).
- תועלת (Benefit - מה יוצא לו מזה).
- תקרת זכוכית (Glass Ceiling - הסרת אשמה על כישלון עבר).

ה. רגע ה"אהה!" (הפריצה הגדולה)
- האם יש רגע ברור שבו הקורא מבין את התובנה החדשה?

ו. בולטים לסירוגין
- האם הבולטים מחליפים בין תועלות לעיגול ⭕ לתועלות למשולש 🔺?

2. הנחיות ספציפיות לעמוד "אודות" 📄

עמוד אודות שונה מעמוד מכירתי רגיל:

**א. השריטה שלנו (לא רק של הלקוח)**
- במקום "אנחנו הכי טובים" -> להראות את הפגיעות/ייחודיות שלנו
- מה אנחנו לא? מה אנחנו לא מבטיחים?
- למה התחלנו? מה הכאב שראינו בשוק?

**ב. אמון באגריגטור (שאלה ספציפית לענף)**
הלקוח שואל: "למה לתת לכם פרטים אם אתם רק מתווכים?"
צריך לענות על:
- מה היתרון שלי שאתם באמצע?
- איך אתם שומרים על האינטרס שלי?
- למה לא ללכת ישירות לבנק/גוף?

**ג. "מי אנחנו" - פגיעות לפני קרדיטים**
- לא רק "ניסיון של X שנים" (זה כללי)
- אלא: מה ראינו שהכאיב לנו? מה גרם לנו להקים את זה?
- הסיפור האישי שגורם לאמון

3.5 כללי SEO לכותרות - חובה! 🎯

**כלל ברזל:** כל כותרת (H1, H2, H3) חייבת לשמור על מילת המפתח המרכזית!

**דוגמאות:**
מילת מפתח: "הלוואה מיידית"

❌ שגוי - מאבד את מילת המפתח:
- "מהי הלוואה מיידית?" -> "בוא נודה באמת - מה מעניין אותך?"
- "יתרונות הלוואה מיידית" -> "למה זה הפתרון הכי טוב?"

✅ נכון - שומר על מילת המפתח + אטומי:
- "מהי הלוואה מיידית?" -> "הלוואה מיידית - מה באמת צריך לדעת?"
- "יתרונות הלוואה מיידית" -> "הלוואה מיידית: היתרונות שלא יספרו לך"
- "איך לקבל הלוואה מיידית?" -> "הלוואה מיידית - הדרך הנכונה להתקבל"

**הנוסחה לכותרת אטומית עם SEO:**
```
[מילת מפתח] + [רגש/שאלה/הבטחה אטומית]
```

**דוגמאות (להשראה בלבד - כל עמוד צריך ניסוח ייחודי משלו!):**

| מילת המפתח של העמוד | דוגמה לכותרת (לא להעתיק!) |
|---------------------|---------------------------|
| הלוואה מיידית | "הלוואה מיידית - מה צריך לדעת לפני?" |
| הלוואה לעסקים | "הלוואה לעסקים - הדרך הנכונה לצמיחה" |
| איחוד הלוואות | "איחוד הלוואות - סוף סוף לנשום" |

⚠️ **חובה:** צור ניסוחים מקוריים שמתאימים **למילת המפתח של העמוד הספציפי**!

3.6 שכתוב FAQ לפי שיווק אטומי ❓

**חובה:** סרוק את אזור ה-FAQ בעמוד וגם את ה-Schema JSON-LD של FAQPage.

### א. איך למצוא את ה-FAQ:
```
grep "FAQPage" בקובץ -> מצא את ה-Schema
grep "faq" או "accordion" בקובץ -> מצא את ה-HTML
grep "<h3" באזור ה-FAQ -> מצא את השאלות
```

### ב. עקרונות שכתוב FAQ אטומי:

**השאלות (Questions):**
- ❌ שאלות ויקיפדיות: "מהי הלוואה מיידית?"
- ❌ שאלות יבשות: "מה הריבית על הלוואה?"
- ✅ שאלות שמדברות לכאב: "קיבלתי סירוב בבנק - יש לי סיכוי?"
- ✅ שאלות שמדברות לפחד: "האם יש סיכון שיתפסו לי את המשכורת?"
- ✅ שאלות שמדברות לתקווה: "תוך כמה זמן הכסף יהיה אצלי?"

**התשובות (Answers):**
- ❌ תשובה כללית: "הריבית נקבעת לפי פרופיל הלווה"
- ✅ תשובה מניסיון: "מניסיוננו, רוב הלקוחות מקבלים ריבית בטווח X-Y%"
- ✅ תשובה עם תקרת זכוכית: "הסיבה שסירבו לך בבנק לא אומרת שאין לך סיכוי - הבנקים בודקים לפי קריטריונים מאוד צרים"
- ✅ תשובה עם B2: "יש גופים שמסתכלים על התזרים הנוכחי שלך, לא על מה שקרה לפני 3 שנים"

**מבנה CPBG לכל תשובה:**
1. **Claim:** התשובה הישירה
2. **Proof:** "ראינו מאות מקרים..." / "לפי הנתונים שלנו..."
3. **Benefit:** מה יוצא לקורא מזה
4. **Glass Ceiling:** למה עד עכשיו לא הצליח (בלי אשמה)

### ג. חובה: סנכרון עם Schema JSON-LD

⚠️ **קריטי:** כל שינוי ב-FAQ חייב להיות זהה ב:
1. ה-HTML הנראה (האקורדיון/רשימה)
2. ה-Schema JSON-LD (FAQPage)

**בדיקה:**
```
grep '"name":' בתוך FAQPage schema -> רשימת השאלות
grep '"text":' בתוך FAQPage schema -> רשימת התשובות
```

### ד. דוגמאות לפני/אחרי (להשראה בלבד - אל תעתיק!):

⚠️ **זכור:** הדוגמאות הבאות הן להמחשת העיקרון בלבד!
- **לא להעתיק** - צור ניסוחים מקוריים שמתאימים **לנושא הספציפי של העמוד**
- הדוגמאות פה הן על "הלוואה מיידית" - אם העמוד שלך על נושא אחר, **התאם!**

**שאלה 1:**
- ❌ לפני: "האם ניתן לקבל הלוואה מיידית עם חוב קיים?"
- ✅ אחרי: "יש לי חוב בבנק - עדיין יש סיכוי להלוואה מיידית?"

**תשובה 1:**
- ❌ לפני: "כן, אך זה מסובך יותר. חברות P2P לעיתים מאשרות..."
- ✅ אחרי: "כן, וזה פחות מסובך ממה שחושבים. מניסיוננו, לקוחות עם חובות קיימים מתקבלים בגופים חוץ-בנקאיים שמסתכלים על יכולת ההחזר הנוכחית שלך - לא על מה שהיה בעבר. הסיבה שהבנק סירב היא לא באשמתך - הם פשוט לא גמישים."

**שאלה 2:**
- ❌ לפני: "מה המסמכים הנדרשים להלוואה מיידית?"
- ✅ אחרי: "הלוואה מיידית - באמת צריך ערימות של מסמכים?"

**תשובה 2:**
- ❌ לפני: "תעודת זהות, 3 תלושי שכר..."
- ✅ אחרי: "הרבה פחות ממה שחושבים. ברוב המקרים מספיק: תעודת זהות, תלושי שכר אחרונים, ודפי בנק. חלק מהגופים אפילו מסתפקים בפחות. מניסיוננו, 80% מהלקוחות מסיימים את התהליך עם מה שכבר יש להם בטלפון."

3.7 כלל ההתאמה: כותרת-פסקה (חובה!) 🔗

> **כלל ברזל:** כל פעם שמשנים כותרת, חובה לבדוק שהפסקה מתחתיה עונה או מרחיבה על הנושא!

**הבדיקה:**
1. קרא את הכותרת החדשה המוצעת
2. קרא את הפסקה הראשונה מתחתיה
3. שאל: "האם הפסקה עונה על השאלה/הנושא שהכותרת מעלה?"
4. אם לא - יש לציין שדרושה התאמת פסקה!

**דוגמה:**
| כותרת חדשה | פסקה מתחת | בעיה | פתרון |
|------------|-----------|------|-------|
| "הלוואה מיידית - למה הבנק לא הפתרון?" | "הלוואה מיידית היא הלוואה שניתנת בזמן קצר..." | הפסקה לא עונה על "למה לא הבנק" | להוסיף משפט פתיחה: "כי בבנק תחכה שבועיים..." ואז להמשיך עם המידע הקיים |
| "מה לא מספרים לך על ריביות?" | "הריבית נקבעת לפי פרופיל הלווה..." | הפסקה לא חושפת "מה לא מספרים" | להוסיף: "הנה הסוד: הריבית שהבנק מציע היא רק נקודת הפתיחה..." |

**הטכניקה הנכונה - הוסף משפט גישור:**
- אל תמחק את הפסקה המקורית!
- הוסף משפט פתיחה שעונה על הכותרת
- המשך עם המידע המקורי

**חשוב:** הכותרת מובילה (אטומית), הפסקה עוקבת - אבל לא מוחקים מידע בעל ערך!

3.8 שמירה על מידע בעל ערך (קריטי!) 🔒

> **כלל על:** שיווק אטומי הוא על **סגנון הכתיבה** - לא על החלפת תוכן אמיתי!

### מה חייבים לשמור:
| סוג מידע | דוגמאות | למה חשוב |
|----------|---------|----------|
| **נתונים פיננסיים** | ריביות, אחוזים, סכומים, טווחי הלוואה | מידע שהגולש צריך לקבל החלטה |
| **טבלאות השוואה** | השוואת גופים, השוואת תנאים | כלי עזר מעשי |
| **דוגמאות מספריות** | "לקוח עם הלוואה של 50,000₪ חסך..." | הוכחה קונקרטית |
| **סיפורי הצלחה** | מקרי בוחן, תיאורי מצב | בניית אמון |
| **מידע רגולטורי** | דרישות חוק, רישיונות, זכויות | חובה משפטית |
| **תהליכים וצעדים** | "שלב 1, שלב 2..." | ערך מעשי לקורא |

### הכלל בפועל:

**❌ לא לעשות - החלפת מידע בשפה ריקה:**
```
לפני: "ריבית ממוצעת: 8-12% לשנה. תקופת החזר: 12-60 חודשים."
אחרי (שגוי): "הריביות? יש כאלה שיפתיעו אותך. התקופות? מותאמות אליך."
```
זה מחליף מידע קונקרטי בשפה ריקה - אסור!

**✅ כן לעשות - שינוי סגנון עם שמירת מידע:**
```
לפני: "ריבית ממוצעת: 8-12% לשנה. תקופת החזר: 12-60 חודשים."
אחרי (נכון): "הריבית? 8-12% לשנה - וזה לפני שמתחילים לדבר. תקופת החזר בין 12 ל-60 חודשים, תלוי במה שנוח לך."
```
שומר על המידע, משנה את הסגנון!

### אם יש סתירה:

> **עיקרון העל:** במקרה של סתירה בין סגנון אטומי לשמירה על מידע - **המידע מנצח!**

- ✅ אפשר לעטוף מידע יבש בשפה אטומית
- ✅ אפשר להוסיף משפטי גישור רגשיים
- ❌ אסור למחוק/להחליף נתונים אמיתיים
- ❌ אסור להפוך טבלאות לטקסט כללי
- ❌ אסור להסיר דוגמאות מספריות

4. מסנן בטיחות YMYL

זהה ביטויים בעייתיים:
❌ "100% הצלחה" -> ✅ "אחוזי הצלחה גבוהים"
❌ "ללא בדיקת אשראי" -> ✅ "קריטריונים גמישים"
❌ "כסף מיידי" -> ✅ "תהליך העברה מהיר"
❌ "מובטח" -> ✅ "הסתברות גבוהה לאישור"

### 🚨 בעיית שקיפות "השירות חינמי" - קריטי!

> **הביטוי "השירות חינמי" מטעה את הגולש!**

**הבעיה:**
- ✅ **מה חינמי:** שירות ההשוואה/הפניה לגופים (מה שאנחנו עושים)
- ❌ **מה לא חינמי:** ההלוואה עצמה - הגופים המלווים גובים ריביות ועמלות
