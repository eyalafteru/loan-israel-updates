## 📋 הוראות הפעלה

1. תייג את הקובץ הזה (`@סוכן יישום אטומי`)
2. תייג את **דוח הביקורת** (`@דוח תיקונים אטומי.md`) - זה הדוח עם הצעות הטקסט המלאות!
3. תייג את קובץ ה-HTML של העמוד (למשל: `@הלוואה מקרן השתלמות.html`)
4. פקודה: "בצע את התיקונים בדוח על קובץ ה-HTML"

**חשוב:** הדוח כולל הצעות טקסט מלאות - השתמש בהן ישירות!

## 📖 קריאת קובץ חכמה - חובה!

קבצי ה-HTML במערכת מכילים CSS/JS מוטמע.
**אסור לקרוא את כל הקובץ!**

### פרוטוקול קריאה:

**שלב 1 - מצא את אזור התוכן:**
```
Grep pattern="</style>" path="[קובץ ה-HTML]"
```
רשום את מספר השורה של ה-`</style>` **האחרון** - התוכן מתחיל כ-100 שורות אחריו.

**שלב 2 - קרא רק את אזור התוכן:**
```
Read path="[קובץ]" offset=[שורה שמצאת + 100] limit=3000
```

**שלב 3 - חיפוש ממוקד לפני עריכה:**
- כותרות: `Grep pattern="<h[1-6]" path="[קובץ]"`
- פסקאות: `Grep pattern="<p class" path="[קובץ]" -A=2`
- הדגשות: `Grep pattern="<strong>" path="[קובץ]"`

### ⛔ אזורים אסורים לקריאה/עריכה:
- **כל מה שלפני `</style>` האחרון** (CSS ו-JavaScript) - מצא את השורה עם Grep ואל תקרא לפניה!
- תגיות `<style>` ו-`<script>`
- namespace classes (כמו `wpc-*-d7k9-*`)

## ⚠️ חשוב מאוד: עריכה על הקובץ הקיים!

**אל תיצור קובץ חדש!** השתמש בכלי **Edit** לערוך את קובץ ה-HTML המקורי ישירות.

### פרוטוקול עריכה בטוחה (Grep-Read-Edit):

**שלב 1 - חפש:**
```
Grep pattern="[טקסט לחיפוש]" path="[קובץ]" -B=2 -A=2
```

**שלב 2 - קרא הקשר:**
```
Read path="[קובץ]" offset=[שורה שמצאת - 5] limit=15
```

**שלב 3 - ערוך בדיוק:**
```
Edit path="[קובץ]" old="[טקסט מדויק]" new="[טקסט חדש]"
```

### למה Edit ולא Write?
- Edit משנה רק את החלק הרלוונטי
- Write דורס את כל הקובץ (מסוכן!)
- Edit שומר על כל העיצוב והמבנה

## 1. פרוטוקול ה"ניתוח" 🏥

### שלב 1: מיפוי דוח → HTML

לפני כל עריכה, מפה את הפריטים מהדוח לתגיות ב-HTML:

| אלמנט בדוח | תגית HTML מטרה | פעולה |
|-----------|----------------|-------|
| **כותרת H1** | `<h1>` או `.hero-title` | החלף טקסט. שמור על `<span>` פנימיים |
| **פתיח/Subtitle** | `.hero-subtitle` או `<p>` ראשון | הזרק את ה"שריטה" וזיהוי הכאב |
| **תוכן ויקיפדי** | `<p>` באזור הפתיח | החלף במשפטים מניסיון |
| **תקרת זכוכית** | `<p>` באזור "למה אנחנו" | הוסף "זה לא באשמתך" |
| **יתרונות/בולטים** | `<ul>`, `<li>`, `.benefit-card` | החלף לסירוגין עיגול/משולש |
| **CTA** | `<button>`, `<a>` | שנה לרגש מטרה |
| **FAQ** | `.accordion-content` | הוסף משפטי ניסיון |

### שלב 2: זיהוי המבנה הייחודי

לפני עריכה, זהה את ה-namespace של העמוד:
- חפש class prefix כמו `wpc-keren-loan-h3k9-*`
- כל תגית חדשה חייבת לשמור על אותו prefix
