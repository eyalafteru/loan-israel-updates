## 📖 קריאת קובץ חכמה - חובה!

קבצי ה-HTML במערכת מכילים CSS/JS מוטמע.
**אסור לקרוא את כל הקובץ! זה מבזבז טוקנים.**

### פרוטוקול קריאה:

**שלב 1 - מצא את אזור התוכן:**
```
Grep pattern="</style>" path="[קובץ ה-HTML]"
```
רשום את מספר השורה של ה-`</style>` **האחרון** - התוכן מתחיל כ-100 שורות אחריו.

**שלב 2 - קרא רק את אזור התוכן:**
```
Read path="[קובץ]" offset=[שורה שמצאת + 100] limit=3000
```

**שלב 3 - חיפוש ממוקד לפי צורך:**
- כותרות: `Grep pattern="<h[1-6]" path="[קובץ]"`
- FAQ: `Grep pattern="FAQPage|accordion|faq" path="[קובץ]"`
- קישורים: `Grep pattern="<a href" path="[קובץ]"`

### ⛔ אזורים אסורים לקריאה:
- כל מה שלפני `</style>` האחרון (CSS ו-JavaScript)
- תגיות `<style>` ו-`<script>`
- namespace classes (כמו `wpc-*-d7k9-*`)

**⚠️ התוכן חייב להיות RTL, רוחב מקסימלי 1200px, ולהתחיל ב-H1!**

```html

[awg postid="3470"]

<!-- המשך התוכן -->

<!-- Schema JSON-LD בסוף, לפני סגירת ה-div -->

</div>
```

**⚠️ חישוב ריבית פריים:**
ריבית פריים = ריבית בנק ישראל + 1.5%
- **דוגמה:** אם ריבית בנק ישראל = 4.0%, אז פריים = 4.0 + 1.5 = **5.5%**

**⚠️ חשוב:**
- כל התוכן חייב להיות עטוף ב-div RTL
- ה-Schema יהיה בתוך ה-div, בסוף
- **משפט הריבית חובה** - מיד אחרי "עודכן לאחרונה"

## ⚛️ עקרונות אטומיים

### הקהל הכפול:
- **העיגול ⭕:** רוצה להצליח, אופטימי
- **המשולש 🔺:** ספקן, מפחד מהסיכון

### מבנה CPBG (לכל תשובה/פסקה):
1. **Claim** - טענה ישירה
2. **Proof** - "מניסיוננו..." / "ראינו מאות מקרים..."
3. **Benefit** - מה יוצא לו מזה
4. **Glass Ceiling** - למה לא הצליח עד עכשיו (בלי אשמה!)

**⚠️ חשוב:** לא לכתוב את המילים Claim/Proof/Benefit בפלט! זה מבנה פנימי.

### שריטה (The Scratch):
זהה על מה הלקוח *לא מוכן לוותר* - ודאג שהפתרון מכבד את זה.

**דוגמאות לעסקים:**
- "העסק צריך לגדול אבל הבנק לא רואה את הפוטנציאל"
- "צריך מימון אבל לא רוצה לשעבד את הבית"
