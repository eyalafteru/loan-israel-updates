## 📖 קריאת קובץ חכמה - חובה!

קבצי ה-HTML במערכת מכילים CSS/JS מוטמע.
**אסור לקרוא את כל הקובץ!**

### פרוטוקול QA סופי:

**שלב 1 - מצא את אזור התוכן:**
```
Grep pattern="</style>" path="[קובץ ה-HTML]"
```

**שלב 2 - סריקת שאריות AI (Blacklist):**
```
Grep pattern="חשוב לציין|ראוי לציין|הינו|הינה|כיצד|מדוע|בסופו של דבר|מרבד של" path="[קובץ]" -i
```

**שלב 3 - בדיקת צפיפות קישורים:**
```
Grep pattern="<p.*<a href.*<a href" path="[קובץ]"
```

**שלב 4 - אימות Atomic Marketing Whitelist:**
```
Grep pattern="מניסיוננו|מהניסיון שלנו|לקוחות רבים" path="[קובץ]"
```

### ⛔ אזורים אסורים:
- **כל מה שלפני `</style>` האחרון**
- תגיות `<style>` ו-`<script>`
- namespace classes

## 🎯 תהליך הבדיקה

### שלב 1: אימות ביצוע שלב 5

קרא את הדוח משלב 5. בדוק מדגמית ב-HTML האם השינויים אכן בוצעו.

---

### שלב 2: סריקת שאריות AI (Blacklist)

חפש ותקן מיידית:

| ביטוי | פעולה |
|-------|-------|
| "חשוב לציין" / "ראוי לציין" / "יש לציין" | מחק |
| "הינו" / "הינה" | שנה ל"הוא"/"היא" |
| "מגוון רחב של" | שנה ל"מגוון" |
| "בעידן הדיגיטלי" | מחק |
| "כעת נעבור" / "נסכם את הדברים" | מחק |
| "בסופו של דבר" | שנה ל"בסוף" |
| "מסע מרתק" / "צלילה לעומק" | מחק |
| "מרבד של" / "נוף פיננסי" | שכתב |

### שלב 3: בדיקת מבנים תחביריים של GPT

| דפוס | בדיקה | תיקון |
|------|-------|-------|
| מקפים מפרידים | "X - שהיא - Y" | הסר מקפים ושכתב |

---

### שלב 4: רשימה לבנה - לא לגעת! (Atomic Marketing)

**אם הסוכן הקודם שינה בטעות - חובה להחזיר!**

| ביטוי מוגן | סיבה |
|-----------|-------|
| "מניסיוננו..." | E-E-A-T |
| "מהניסיון שלנו..." | E-E-A-T |
| "לקוחות רבים..." | Social Proof |
| "ניתן ל..." (בהקשר טכני) | תקין |
| "אנו" (בהקשר רשמי) | תקין |

---

### שלב 5: בדיקת צפיפות קישורים (Gatekeeper)

**הבדיקה הסופית לפני העלייה לאוויר!**

| חוק | בדיקה | תיקון |
|-----|-------|-------|
| מקסימום 2 קישורים לפסקה | `Grep pattern="<p.*<a.*<a.*<a"` | הסר או פצל |
| מרחק מינימלי 3 מילים | בדוק קישורים צמודים | הרחק או מחק |

**הערה:** אל תספור CTA (כפתורים, WhatsApp, טלפון).
