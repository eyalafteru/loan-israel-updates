## 📖 קריאת קובץ חכמה - חובה!

קבצי ה-HTML במערכת מכילים CSS/JS מוטמע בשורות 1-10000.
**אסור לקרוא את כל הקובץ! זה מבזבז טוקנים ומאט את התהליך.**

### פרוטוקול קריאה:

**שלב 1 - מצא את אזור התוכן:**
```
Grep pattern="</style>" path="[קובץ ה-HTML]"
```
רשום את מספר השורה - התוכן מתחיל כ-100 שורות אחרי.

**שלב 2 - קרא רק את אזור התוכן:**
```
Read path="[קובץ]" offset=[שורה שמצאת + 100] limit=3000
```

**שלב 3 - חיפוש ממוקד לריביות:**
```
Grep pattern="[0-9]+\.?[0-9]*%" path="[קובץ]" -B=1 -A=1
Grep pattern="פריים" path="[קובץ]" -B=2 -A=2
Grep pattern="<table" path="[קובץ]" -A=20
```

### ⛔ אזורים אסורים לקריאה:
- כל מה שלפני `</style>` האחרון (CSS ו-JavaScript)
- תגיות `<style>` ו-`<script>`
- namespace classes (כמו `wpc-*-d7k9-*`)

### שלב 0 - זיהוי הריבית הנוכחית בתוכן (קריטי!)

לפני שאתה מזהה שינויים, **חפש בתוכן** את משפט הגילוי:

**📍 מיקום המשפט בעמוד:**
המשפט תמיד מופיע **מיד אחרי** השורה:
```
עודכן לאחרונה: [תאריך]
```

המשפט מתחיל ב-💡 ונראה כך:
```
💡 הריביות בתוכן מעודכנות לפי ריבית פריים X% (ריבית בנק ישראל Y%, נכון ל-[חודש שנה])
```

**📍 מבנה HTML - גרסאות שונות (חשוב!):**

המבנה יכול להשתנות בין דפים. דוגמאות:

**גרסה 1 - עם `<p>` ו-`<br>`:**
```html
<p>עודכן לאחרונה: [current_date format="F Y" hebrew="true"]<br>
💡 הריביות בתוכן מעודכנות לפי ריבית פריים X% (ריבית בנק ישראל Y%, נכון ל-[חודש שנה])<br>
</p>
```

**גרסה 2 - עם `<strong>` ו-`<em>`, ללא `<br>`:**
```html
<strong>עודכן לאחרונה:</strong> [current_date format="F Y" hebrew="true"]
<em style="color: #666; font-size: 0.95em;">הריביות בתוכן מעודכנות לפי ריבית פריים X% (ריבית בנק ישראל Y%, נכון ל-[חודש שנה])</em>
```

**⚠️ כללים קריטיים - מבנה השורות:**
- ✅ רשום בדוח את המבנה המדויק שמצאת (עם כל התגיות והעיצוב)
- ✅ התאריך הוא shortcode דינאמי `[current_date...]` - לא לגעת בו!
- ✅ רק הריביות והתאריך העברי במשפט הגילוי צריכים עדכון
- ✅ שמור במדויק את מבנה ה-HTML המקורי (תגיות, `<br>`, `style`, וכו')

**חיפוש:**
```
Grep pattern="הריביות בתוכן מעודכנות" path="[קובץ]" -B=3 -A=1
```

**⚠️ חשוב: זה שלב ניתוח בלבד - אל תערוך כלום! אל תתקן! רק רשום בדוח מה צריך לעדכן.**

**⚠️ אל תשנה שום דבר בקובץ HTML! אל תוסיף בולד, אל תוסיף ירידות שורה, אל תשנה עיצוב!**

**אם נמצא:**
- ✅ `old_prime` = X%
- ✅ `old_boi` = Y%
- ✅ `detection_source` = "rate_disclosure_line"
- ✅ `confidence` = "high"

**אם לא נמצא (תוכן ישן ללא הצהרה):**
- ⚠️ הסק מהטבלאות: מצא את הריבית הנמוכה ביותר בעמודת "ריבית"
- ⚠️ הסק מהפסקאות: חפש "החל מ-X%" או "ריבית X%"
- ⚠️ `detection_source` = "inferred_from_content"
- ⚠️ `confidence` = "medium"
- ⚠️ הוסף warning: "לא נמצאה הצהרת ריביות - הסקתי מהתוכן"

### שלב 0.5 - זיהוי ריביות לא חוקיות

**⚖️ תקרת הריבית החוקית בישראל: 19.5%**

לפני שאתה מעדכן ריביות רגילות, **חפש ריביות לא חוקיות** בתוכן:

#### מה לחפש:
- ריבית בודדת מעל **19.5%** (כמו 22%, 25%, 30%, 35%)
- דוגמאות עם ריביות לא חוקיות
- חישובי חיסכון/פיצויים שקשורים לריביות לא חוקיות

#### מה לעשות:
1. זהה את הריבית הלא חוקית
2. עדכן ל-19.25% (התקרה החוקית המקסימלית)
3. עדכן חישובים:
   - חיסכון שנתי = סכום × (ריבית_ישנה - 19.5%)
   - פיצוי כפול = חיסכון × 2

#### נוסחאות עדכון:
```
חיסכון_שנתי = סכום_הלוואה × (ריבית_מוצעת - 19.5) / 100
פיצוי_כפול = חיסכון_שנתי × 2
```

### שלב 1 - קרא את הריביות החדשות

מהנתונים שקיבלת חלץ:
- ✅ ריבית פריים: הערך שקיבלת
- ✅ ריבית בנק ישראל: פריים - 1.5%
- ✅ חשב את ההפרש: new_prime - old_prime

### שלב 3 - אסטרטגיית עדכון חכמה

**כלל אצבע:** אם פריים השתנה ב-ΔX%, כל הריביות משתנות ב-ΔX%

#### דוגמה:
```
פריים ישן: 6.0%
פריים חדש: 5.75%
הפרש: -0.25%

טווח ישן: "6.5%-10%"
טווח חדש: "6.25%-9.75%" (כל קצה -0.25%)

נוסחה: "פריים + 1% (7.0%)"
חדש: "פריים + 1% (6.75%)" (הנוסחה נשארת, הערך משתנה)
```

#### קצה-מקרים:

1. **טווח ריביות:** "X%-Y%"
   - רוחב הטווח נשאר קבוע
   - שני הקצוות משתנים באותו ΔX%

2. **נוסחה + ערך:** "פריים + 1% (6.75%)"
   - הנוסחה לא משתנה
   - הערך בסוגריים משתנה
   - **חשוב:** אם אין סוגריים עם ערך → לא מעדכנים! (רק "פריים + 1%" ללא ערך)

3. **ריבית + תאריך:** "6% (יולי 2025)"
   - גם הריבית וגם התאריך משתנים!
   - דוגמה: "6% (יולי 2025)" → "5.75% (ינואר 2026)"

4. **ריביות קבועות:**
   - אם כתוב "ריבית קבועה X%" → לא מעדכנים
   - רק ריביות תלויות-פריים מתעדכנות

5. **דוגמאות מחושבות:**
   - אם יש "חיסכון של X ש״ח" → צריך לחשב מחדש!
   - אם יש "תשלום חודשי Y ש״ח" → צריך לחשב מחדש!

## 🚨 כללי החרגות - מתי לא לעדכן ריבית

### ❌ אל תעדכן ריבית במקרים הבאים:

#### 1. ריבית קבועה מפורשת:
```
"ריבית קבועה 4%" → לא לעדכן ❌
"ריבית קבועה לכל התקופה" → לא לעדכן ❌
```

#### 2. ריביות של מוצרים אחרים (לא הלוואות):
```
"כרטיס אשראי בריבית 12%" → לא לעדכן ❌
"משיכת מזומנים 15%" → לא לעדכן ❌
"החזר צ'קים 18%" → לא לעדכן ❌
"ריבית דביטורית 8%" → לא לעדכן ❌
"עו"ש פלוס 10%" → לא לעדכן ❌
```

#### 3. ריביות גבוהות מדי שלא צמודות לפריים (>15% אבל לא דוגמאות):
```
"ריבית 18% בבנק" → כנראה לא צמוד לפריים, לא לעדכן ❌
"ריבית 20% לעסק" → בטח לא צמוד לפריים, לא לעדכן ❌
```

**⚠️ חריג חשוב - ריביות לא חוקיות בדוגמאות:**
```
"דני קיבל הצעה של 30,000 ש"ח ב-35% ריבית" → כן לעדכן! ✅
```
אם הריבית היא **חלק מדוגמה** על ריבית לא חוקית (מעל 19.5%) - **כן** צריך לעדכן!

#### 4. נוסחה בלבד ללא ערך מחושב:
```
"פריים + 1.5%" → לא לעדכן ❌ (הנוסחה לא משתנה!)
"פריים + 3.5%" → לא לעדכן ❌ (אין ערך מחושב לעדכן)
```

#### 5. אין קונטקסט של הלוואה:
אם אין מילים כמו "הלוואה", "מימון", "אשראי", "משכנתא" בקרבת מקום:
```
"ריבית 8%" (ללא קונטקסט) → לא לעדכן ❌
```

### ✅ עדכן ריבית רק במקרים הבאים:

#### 1. מופיע "פריים" או "בנק ישראל":
```
"פריים + 1%" → עדכן ✅
"ריבית בנק ישראל + 2%" → עדכן ✅
"מבוסס על ריבית הפריים" → עדכן ✅
```

#### 2. טווח ריבית הגיוני (4%-15%):
```
"ריבית 6%-10%" → כנראה צמוד לפריים, עדכן ✅
"הלוואה בריבית 7.5%" → בטווח הגיוני, עדכן ✅
```

#### 3. בטבלת הלוואות או בפסקה על הלוואות:
אם הקונטקסט כולל:
- "הלוואה לכל מטרה"
- "הלוואה לעובדים"
- "מימון לרכישת"
- "אשראי בנקאי"

→ כנראה צמוד לפריים, עדכן ✅
