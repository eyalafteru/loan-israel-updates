## 🚨 כלל קריטי: התוכן חייב להיראות אנושי לחלוטין!

### אסור בשום אופן להסגיר AI:

**אל תשתמש ב:**
- הערות כמו `<!-- Atomic Change: ... -->` בקוד הסופי
- מילים נפוצות מדי אצל AI: "מדהים", "פנטסטי", "מושלם", "בהחלט", "אכן"
- משפטים שמתחילים ב: "זה חשוב לציין ש...", "ראוי לציין ש...", "כדאי לדעת ש..."
- רשימות עם בדיוק 3 או 5 פריטים כל פעם (גוון את המספר!)
- פסקאות באורך זהה אחת אחרי השנייה
- מבנים חזרתיים כמו: "[X] זה לא רק [Y] - זה גם [Z]"

**כן כתוב:**
- משפטים בגבהים שונים (קצרים וארוכים לסירוגין)
- שפה יומיומית שאנשים באמת מדברים ("הלב לא נותן", "חופרים לך")
- חוסר סימטריה מכוון - לא הכל "מסודר" מדי
- טעויות קטנות בכוונה כמו: "לא?" במקום "נכון?"
- ביטויים ישראליים אותנטיים: "תכלס", "יאללה", "בסוף", "לא הולך לקרות"

### טכניקות להסוואת AI:

1. **אל תסמן שינויים בקוד** - פשוט בצע אותם ללא הערות
2. **שבור תבניות** - אם יש 4 כרטיסים, תן כותרת באורך שונה לכל אחד
3. **הימנע מקלישאות** - במקום "חוסכים אלפי שקלים" כתוב "מפסיקים לזרוק כסף לפח"
4. **הוסף "רעש אנושי"** - משפט שאלה, מילת מעבר לא צפויה, קריאת ביניים

## ⚠️ כלל העל: שמור על מבנה הקוד!

### מותר לך:
- לשנות **תוכן טקסט** בתוך תגיות
- לפצל `<p>` אחד לשני `<p>` עם אותם classes
- להוסיף תגית `<p>` או `<span>` חדשה בסגנון זהה לסביבה

### אסור לך בשום אופן:
- לשנות `class`, `id`, או `style` attributes
- לשנות את היררכיית ה-DOM
- למחוק תגיות מבניות (`<section>`, `<div>`)
- לשנות CSS או JavaScript

## 1. פרוטוקול ה"ניתוח" 🏥

### שלב 1: מיפוי דוח → HTML

לפני כל עריכה, מפה את הפריטים מהדוח לתגיות ב-HTML:

| אלמנט בדוח | תגית HTML מטרה | פעולה |
|-----------|----------------|-------|
| **כותרת H1** | `<h1>` או `.hero-title` | החלף טקסט. שמור על `<span>` פנימיים |
| **פתיח/Subtitle** | `.hero-subtitle` או `<p>` ראשון | הזרק את ה"שריטה" וזיהוי הכאב |
| **תוכן ויקיפדי** | `<p>` באזור הפתיח | החלף במשפטים מניסיון |
| **תקרת זכוכית** | `<p>` באזור "למה אנחנו" | הוסף "זה לא באשמתך" |
| **יתרונות/בולטים** | `<ul>`, `<li>`, `.benefit-card` | החלף לסירוגין עיגול/משולש |
| **CTA** | `<button>`, `<a>` | שנה לרגש מטרה |
| **FAQ** | `.accordion-content`, תשובות | הוסף משפטי ניסיון |

### שלב 2: זיהוי המבנה הייחודי

לפני עריכה, זהה את ה-namespace של העמוד:
- חפש class prefix כמו `wpc-keren-loan-h3k9-*`
- כל תגית חדשה חייבת לשמור על אותו prefix

## 2. הזרקת תוכן אטומי - לפי סדר עדיפויות

### עדיפות 1: השריטה (The Scratch) 🔥

**מיקום:** פתיח/Subtitle מתחת ל-H1

**לפני (ויקיפדי):**
```html
<p class="wpc-hero-subtitle">הלוואה מקרן השתלמות היא אחת מדרכי המימון המשתלמות ביותר...</p>
```

**אחרי (אטומי - בלי הערות!):**
```html
<p class="wpc-hero-subtitle">צריך כסף אבל הלב לא נותן לך לשבור את הקרן שבנית שנים? אתה לא לבד.</p>
```

### עדיפות 2: תקרת הזכוכית (Glass Ceiling) 🔥

**מיקום:** פסקה שנייה או אזור "למה אנחנו"

**פעולה:** מצא `<p>` קרוב והוסף אחריו פסקה חדשה באותו סגנון:

```html
<p class="wpc-content-text">
    לא ידעת על זה עד היום? זה לא באשמתך. 
    הבנקים מעדיפים שתיקח הלוואה יקרה אצלם, והקרנות? הן לא ממש מתלהבות לספר.
</p>
```

### עדיפות 3: תוכן מניסיון (Experience-Based) 🔥

**מיקום:** כל מקום עם הגדרות כלליות

**לפני:**
```html
<p>קרן השתלמות היא מכשיר חיסכון ייחודי המיועד לשכירים...</p>
```

**אחרי (שים לב לשפה הטבעית):**
```html
<p>יודע מה הבעיה? רוב האנשים שמגיעים אלינו בכלל לא יודעים שאפשר לקחת הלוואה מהקרן. ככה זה - הדברים הכי משתלמים הם גם הכי פחות מפורסמים.</p>
```

### עדיפות 4: יתרונות לסירוגין (עיגול/משולש)

**מיקום:** כרטיסי יתרונות, רשימות

**עיקרון:** להחליף בין תועלת לעיגול ⭕ לתועלת למשולש 🔺

```html
<div class="wpc-benefit-card">
    <h3>5.15% ריבית. זהו.</h3>
    <p>נמוך מכל הלוואה בנקאית - ובפער גדול</p>
</div>

<div class="wpc-benefit-card">
    <h3>בלי קאצ'ים</h3>
    <p>מה שסגרת זה מה שתשלם. בלי עמלות שצצות פתאום</p>
</div>
```

### עדיפות 5: קריאה לפעולה (CTA)

**לפני (גנרי):**
```html
<button>בדוק זכאות עכשיו</button>
```

**אחרי (רגש מטרה):**
```html
<button>בוא נראה כמה יש לך בקרן - בלי לאבד שקל</button>
```

## 4. טיפול במקרים מיוחדים

### אם הדוח אומר "הוסף תקרת זכוכית" אבל אין placeholder:

**פעולה:** שכפל את ה-`<p>` הקרוב ביותר (עם כל ה-classes שלו) והוסף את הטקסט החדש:

```html
<p class="wpc-section-text" style="...existing styles...">
    [טקסט קיים]
</p>
<p class="wpc-section-text" style="...same styles...">
    [טקסט תקרת זכוכית - ללא הערות בקוד!]
</p>
```

### אם יש inline styles:

**פעולה:** שמור עליהם בדיוק כמו שהם, גם אם הם ארוכים.

### אם הדוח מבקש לשנות FAQ:

**פעולה:** שנה רק את תוכן התשובה, הוסף משפט מניסיון בסוף:

```html
<p>תהליך האישור אורך 3-5 ימי עסקים. <strong>אצלנו? רוב הלקוחות מקבלים אישור תוך 3 ימים.</strong></p>
```
