### 3. עדכון Schema Markup (שורות 277-282)

- **מיקום:** `annualPercentageRate` ב-FinancialProduct schema
- **ערך ישן:**
```json
"annualPercentageRate": {
    "@type": "QuantitativeValue",
    "minValue": "6",
    "maxValue": "16",
    "unitText": "%"
}
```
- **ערך חדש:**
```json
"annualPercentageRate": {
    "@type": "QuantitativeValue",
    "minValue": "5.75",
    "maxValue": "15.75",
    "unitText": "%"
}
```

**הסבר:** הריביות בטבלה הן נוסחאות ביחס לפריים. עם פריים 5.5%:
- מינימום: עמותות סיוע בפריים + 0% = 5.5%, אך ה-schema מציין 6%, נניח שהחישוב היה פריים + 0.25% (5.75% + 0.25% = 6%). עם פריים חדש: 5.5% + 0.25% = 5.75%
- מקסימום: חברות חוץ-בנקאיות פריים + 10% = 15.75% (עם פריים ישן היה 15.75%, עכשיו יהיה 15.5%).
- **תיקון:** minValue: 6 → 5.75, maxValue: 16 → 15.75 (ירידה של 0.25% בשני הקצוות)

## ⚠️ אזהרות והערות

1. **הטבלה מכילה רק נוסחאות:** כל הריביות בטבלה הן ביחס לפריים (כמו "פריים + 3-5%") ללא ערכים מחושבים בפועל. לפי ההנחיות, נוסחאות בלבד לא מתעדכנות.

2. **מומלץ לשקול הוספת ערכים מחושבים:** כדאי לשקול אם להוסיף בעתיד ערכים מחושבים בסוגריים לטבלה (למשל: "פריים + 3-5% (8.5%-10.5%)").

3. **Schema Markup:** יש לוודא שהערכים ב-schema תואמים את הטווח האמיתי המוצג בתוכן.
